<objective>
Build the `GestaoFerias_VacationApproval` automated cloud flow in Power Automate following the
blueprint at `docs/Flow-Blueprint-VacationApproval.md`. This flow handles the full vacation request
approval lifecycle: trigger on item creation → status guard → approval routing → balance deduction
(if approved) → dual-channel notifications (Teams + Email) in PT-BR.

**This is a browser-guided build** — the flow is constructed interactively in the Power Automate
designer at make.powerautomate.com. Each task below corresponds to a step in the designer.
</objective>

<must_haves>
- Flow named `GestaoFerias_VacationApproval` exists in Power Automate environment
- Trigger is "When an item is created" on Solicitacoes_Ferias list
- Status guard prevents re-processing of non-PENDING items
- Approval request reaches the correct manager (from Email_Aprovador field)
- Approved path: request status updated to APPROVED + balance deducted + employee notified via Teams AND Email
- Rejected path: request status updated to REJECTED + employee notified with rejection reason via Teams AND Email
- All user-facing text is in PT-BR
- Dates formatted as dd/MM/yyyy
</must_haves>

- docs/Flow-Blueprint-VacationApproval.md (Step 4)
</read_first>
<action>
Add a **Condition** action after GetEmployeeDetails:

- **Left operand:** `@{triggerOutputs()?['body/Status/Value']}`

Save the flow.
</action>
